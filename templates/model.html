### Predicting the Probability of default -"GiveMeSomeCredit" from <a href='https://www.kaggle.com/c/GiveMeSomeCredit/overview'>Kaggle</a> This application provides REST interface to predict the probabilities of defaults. It handles only POST request (GET request are not allowed) at the [/predict](predict) route #### Model Summary * The model is based on the Kaggle competition dataset used for "GiveMeSomeCredit" competition. The competition main page is found <a href='https://www.kaggle.com/c/GiveMeSomeCredit/overview'>here</a>. * `etl.py` can be used to automate the ETL to extract the data and save it in the local environment. * Model training can be found in the `notebooks/train-credit.ipynb`. * I used `scikit-learn` Random Forecast Classifier with GridSearchCV method to find the best fitting model. * The highest rank model in the competition had a score of 0.86. My model RFC Kaggle score was 0.859. <img src="/static/images/prediction.png" alt="Test Data Prediction"> * The model predicts the probability of a customer (or application for a loan) defaulting over the next 2 years * The more precise definition of default is if a "Person experienced 90 days past due delinquency or worse." It is represented by a response variable **`SeriousDlqin2yrs`** <br><br> #### Data Dictionary | Variable Name | Description | Type | |-|-|-| | SeriousDlqin2yrs | Person experienced 90 days past due
